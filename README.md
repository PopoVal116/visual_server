## visual_server

Серверная часть системы визуального программирования.  
Принимает запросы от Android-клиента, обрабатывает данные, хранит информацию в базе и отображает результаты в удобном графическом интерфейсе.

## Возможности

- Приём и обработка сообщений от Android-клиента в реальном времени
- Визуализация данных и графиков
- Хранение информации в PostgreSQL
- Интерактивный графический интерфейс на базе ImGui + ImPlot
- Масштабируемая клиент-серверная архитектура

## Стек технологий

- **Язык**: C++17
- **Сетевое взаимодействие**: ZeroMQ (libzmq)
- **Графический интерфейс**: Dear ImGui + ImPlot
- **Окно и рендеринг**: SDL2 + OpenGL + GLEW
- **База данных**: PostgreSQL
- **Сборка**: CMake

## Архитектура

Сервер состоит из:
- Основного цикла обработки ZeroMQ-сообщений
- Графического интерфейса для мониторинга и визуализации
- Модуля работы с базой данных
- Системы логирования и обработки команд от клиента

## Сборка и запуск

```bash
# Клонирование с субмодулями (обязательно!)
git clone --recursive https://github.com/PopoVal116/visual_server.git
cd visual_server
```

# Переходим в папку сервера
```
cd cpp_server
```

# Сборка
```
mkdir build && cd build
cmake ..
make -j4
```
Запуск
```Bash
./server
```

Сервер запускается с графическим окном и начинает прослушивать подключения от Android-клиента по ZeroMQ.

## Связь с клиентом
Сервер работает в паре с Android-приложением visual_programming.
Обмен данными осуществляется через ZeroMQ. Клиент отправляет визуальные конструкции и данные (включая геолокацию), сервер их обрабатывает и возвращает результат.
